POSTMORTEM DIAGNOSIS
====================

## Run summary
- Run: runs/0316_01_v2
- Duration: 5705 updates
- Best reward: ~47.88 at update 5690
- Final reward: ~45.06

## What worked
- The agent successfully learned to survive and grow to a significant length (avg snake size ~73).
- The training was stable, with the agent consistently achieving high rewards and long episode lengths (avg length ~1700).

## What failed
- The agent still relies heavily on the zigzagging "reward hacking" strategy instead of dynamic pathfinding.
- The intended architectural changes (individual channels for head, body, foot) were not fully implemented; there is no dedicated channel for the tail/foot.

## Behavior vs expectation
- Expected: The snake should infer the direction of travel and the length of the body, and tell which block would free itself fastest. With a deeper network layer and individual channels per element (head, body, foot), the signal should be cleaner. Together, these changes should solve the reward hacking issue where the snake zigzags.
- Observed: The agent exhibits a highly conservative, repetitive zigzagging pattern (resembling a Hamiltonian cycle). It prioritizes absolute safety over efficiency, only eating the food when its exhaustive, sweeping path happens to cross it.
- Gap: The zigzagging persists because the explicit tail channel is missing, preventing the network from identifying the exact block that will free itself next. Additionally, the body gradient signal for the tail approaches zero as the snake grows, making it virtually indistinguishable from empty space for a CNN using ReLU activations.

## Next experiments (max 3)
1. Add a dedicated binary channel for the tail/foot to the observation space and update the model to accept 4 channels. This will explicitly tell the network which block will free itself next, regardless of the snake's length.
2. Implement frame stacking (e.g., stacking the last 2-3 frames) or add an explicit direction channel to help the agent infer the direction of travel and plan dynamic routes.
3. Tune the reward function to increase the step penalty relative to the death penalty, encouraging the agent to take more efficient paths to the food rather than relying on the safe, long zigzag pattern.

## Recommendation
Adjust architecture to include a dedicated tail channel and consider adding frame stacking or direction channels.
