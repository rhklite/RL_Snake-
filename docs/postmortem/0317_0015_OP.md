POSTMORTEM DIAGNOSIS
====================

## Run summary
- Run: 0316_21_bfs-reachability (W&B: tough-bird-25)
- Duration: 391 / 30,517 updates (1.3% complete), manually stopped
- Total timesteps: ~12.8M (391 updates x 256 envs x 128 steps)
- Best return: -1.22 at update 386
- Final return: -1.33
- SPS: ~1,750 (stable after warmup)

## What worked
- Numba JIT BFS compilation recovered substantial SPS: ~1,750 vs the predicted ~800 for pure-Python BFS. The Numba optimization delivered a ~2x improvement over the naive estimate, though still well below the ~8,000 SPS of the previous 3-channel run without BFS.
- Memory guard at 98% prevented OOM kills during this run. The process did not terminate from memory pressure.

## What failed

### 1. Complete failure to learn food-seeking
Snake size remained at 1.0-1.1 across all 391 updates. The agent never consistently ate food. The return plateaued at ~-1.43 by update 30 and barely improved over the next 360 updates. The best return of -1.22 at update 386 still reflects an agent that dies in ~17 steps without eating.

### 2. Premature entropy collapse
Entropy dropped from 1.386 (near-uniform over 4 actions) to ~0.69 by update 98 and continued declining to 0.55 by update 391. The policy locked onto a single dominant action (move upward based on video analysis). With ent_coef=0.03, the entropy bonus was insufficient to maintain exploration against the reward signal that slightly favors a straight-line path toward food.

### 3. Hard plateau at episode length 16.0
Episode length crashed from ~95 to 16.0 by update 30 and remained flat at exactly 16.0 for ~340 consecutive updates. On a 32x32 grid, 16 steps is the distance from center to wall -- the agent learned to walk in a straight line and die. This is a degenerate local optimum where the distance shaping reward is marginally positive for moving toward food but the agent converges on a single direction before discovering that reaching food yields a larger reward.

### 4. Clip fraction near zero for most of training
Clip fraction was essentially 0.0 from update 1 through ~370, then jumped to 0.04 at the end. This means PPO's clipping constraint was never active -- policy updates were too small to trigger it. Combined with the low entropy, this suggests the policy gradient signal was extremely weak and updates were too conservative to escape the local minimum.

## Behavior vs expectation

- Expected: Snake avoids enclosed regions, survives beyond length 80, targets 150+ (15% grid fill), body deaths decrease
- Observed: Snake moves in a straight line upward for ~16 steps, dies by wall collision at length 1. 99.97% wall deaths. Zero body deaths (vacuously -- no body exists). No spatial reasoning, no food-seeking, no staircase pattern.
- Gap: The run achieved none of the hypothesis goals. The BFS reachability channel was never tested because the agent never grew long enough for body topology to matter. The root cause is not the observation design but a training dynamics failure: premature policy collapse into a degenerate single-action attractor before food-seeking behavior emerged.

## Comparison to previous run (0316_01_v2)

The 3-channel body-gradient run achieved best return 49.9, snake length 83, and learned food-seeking behavior. This 4-channel BFS run regressed catastrophically. Key differences:
- MaxPool removal increased the CNN's flatten dimension, changing the learning dynamics
- The 1x1 bottleneck (128 to 4 channels) is an aggressive information compression that may discard features needed for early learning
- The BFS channel provides no useful signal at snake length 1 (all cells are reachable with near-identical values), so it adds noise without helping early-stage learning
- SPS dropped from ~8,000 to ~1,750, meaning the same wall-clock time yields ~4.5x fewer gradient updates

## Root cause analysis

The primary failure mode is **premature policy collapse**, not an observation design flaw. The agent converged on a single-action policy before discovering that food-seeking yields reward. Contributing factors:

1. **Entropy coefficient too low for this architecture**: 0.03 was sufficient for the 3-channel MaxPool architecture but insufficient here. The larger observation space and different architecture landscape may require higher entropy regularization to maintain exploration.
2. **Distance shaping too weak**: alpha=0.01 provides tiny per-step rewards (~0.01 per step closer to food). On a 32x32 grid, the cumulative shaping reward over a 16-step episode is ~0.16, while the wall-death penalty dominates at -1.5. The agent learns "don't die" but not "seek food."
3. **BFS channel is noisy at length 1**: When the snake is length 1, the BFS reachability map is nearly uniform (all cells reachable, values in 0.96-1.0 range). This provides almost no gradient signal. The agent must first learn to eat food (grow longer) before the BFS channel becomes informative -- a chicken-and-egg problem.
4. **Architecture regression**: Removing MaxPool and adding a 1x1 bottleneck simultaneously changed two aspects of the CNN. It's unclear which (or both) contributed to the regression. The bottleneck compresses 128 feature channels to 4, which may be too aggressive for early feature learning.

## Next experiments (max 3)

1. **Restore the previous working architecture (3-channel MaxPool) and train longer as a baseline.** The 0316_01_v2 run proved the 3-channel body-gradient observation works. Revert to that architecture, confirm it reproduces the best return of ~50 and snake length ~80, and establish a solid baseline before adding complexity. This isolates whether the regression is from the architecture change or the observation change.

2. **Add BFS channel to the working MaxPool architecture (4-channel with MaxPool).** Keep the MaxPool layers from the working architecture and add the BFS reachability as a 4th channel. This tests the observation hypothesis without the architecture regression. If the snake reaches length 80+ and the BFS channel helps it survive longer, the observation design is validated.

3. **Increase entropy coefficient to 0.05-0.10 and increase dist_shaping_alpha to 0.05.** The premature entropy collapse is the proximate cause of failure. Higher ent_coef forces continued exploration. Higher distance shaping makes food-seeking more rewarding relative to the death penalty, helping the agent discover food before collapsing to a wall-death attractor. Test this with the current 4-channel no-MaxPool architecture to see if the architecture can learn given sufficient exploration.

## Recommendation

Restore the proven 3-channel MaxPool architecture as a working baseline, then incrementally add the BFS channel. The current run changed too many variables simultaneously (observation, pooling, bottleneck) and regressed catastrophically. Isolate the variables.
